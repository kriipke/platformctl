# 🔍 ApplicationSet Query Commands - Curl Examples

## 📋 **1. CONTEXTOPS API QUERIES**

### Basic Health & System Status
```bash
# ContextOps system health (shows ArgoCD integration is active)
curl -s -u admin:admin http://45.55.118.17/health

# List all ContextOps contexts (shows GitOps monitoring capabilities)
curl -s -u admin:admin http://45.55.118.17/api/v1/contexts | jq '.contexts[] | select(.spec.gitops.monitoring.applicationSets == true)'
```

### GitOps API Endpoints (may require customer context)
```bash
# GitOps health overview
curl -s -u admin:admin http://45.55.118.17/api/v1/gitops/health/overview

# GitOps contexts status  
curl -s -u admin:admin http://45.55.118.17/api/v1/gitops/contexts/status

# Specific context GitOps status
curl -s -u admin:admin http://45.55.118.17/api/v1/gitops/contexts/payment-service-dev/status
```

## 📋 **2. KUBERNETES API QUERIES (Direct kubectl)**

### ApplicationSet Information
```bash
# Get ApplicationSet with ContextOps monitoring labels
kubectl get applicationsets -n contextops -l contextops.io/monitored=true -o json | jq '.items[0].metadata'

# ApplicationSet detailed status and conditions
kubectl get applicationsets -n contextops contextops-sample-apps -o json | jq '.status'

# ApplicationSet generator configuration (shows multi-environment setup)
kubectl get applicationsets -n contextops contextops-sample-apps -o json | jq '.spec.generators[0].list.elements'
```

### Generated Applications
```bash
# All applications generated from our ApplicationSet
kubectl get applications -n contextops -l contextops.io/applicationset=contextops-sample-apps -o json | jq '.items[] | {name: .metadata.name, customer: .metadata.labels."contextops.io/customer", environment: .metadata.labels."contextops.io/environment", monitored: .metadata.labels."contextops.io/monitored"}'

# Specific application details
kubectl get application acme-corp-dev-sample -n contextops -o json | jq '{name: .metadata.name, labels: .metadata.labels, sync_status: .status.sync.status, health: .status.health.status}'
```

## 📋 **3. KUBERNETES REST API VIA CURL (with kubectl proxy)**

Start kubectl proxy first:
```bash
kubectl proxy --port=8080 &
```

### ApplicationSet Queries
```bash
# Get ApplicationSet metadata and labels
curl -s http://localhost:8080/apis/argoproj.io/v1alpha1/namespaces/contextops/applicationsets/contextops-sample-apps | jq '.metadata.labels'

# Get ApplicationSet status
curl -s http://localhost:8080/apis/argoproj.io/v1alpha1/namespaces/contextops/applicationsets/contextops-sample-apps | jq '.status'

# Get ApplicationSet spec (generators, template, etc.)
curl -s http://localhost:8080/apis/argoproj.io/v1alpha1/namespaces/contextops/applicationsets/contextops-sample-apps | jq '.spec'
```

### Application Queries  
```bash
# List all applications generated by our ApplicationSet
curl -s "http://localhost:8080/apis/argoproj.io/v1alpha1/namespaces/contextops/applications?labelSelector=contextops.io/applicationset=contextops-sample-apps" | jq '.items[] | {name: .metadata.name, environment: .metadata.labels."contextops.io/environment"}'

# Get specific application
curl -s http://localhost:8080/apis/argoproj.io/v1alpha1/namespaces/contextops/applications/acme-corp-dev-sample | jq '{name: .metadata.name, labels: .metadata.labels, destination: .spec.destination, source: .spec.source}'
```

## 📋 **4. INTEGRATION STATUS QUERIES**

### ContextOps ArgoCD Integration Configuration
```bash
# Check if ArgoCD is enabled in ContextOps
kubectl get configmap contextops-integration-config -n contextops -o json | jq '.data."argocd.enabled"'

# Get ArgoCD server URL configured in ContextOps  
kubectl get configmap contextops-integration-config -n contextops -o json | jq '.data."argocd.server.url"'

# Get all ArgoCD-related configuration
kubectl get configmap contextops-integration-config -n contextops -o json | jq '.data | with_entries(select(.key | contains("argocd")))'
```

### Environment Variables in ContextOps Services
```bash
# Check ArgoCD integration variables in gateway
kubectl exec -n contextops $(kubectl get pods -n contextops -l app=contextops-gateway -o jsonpath='{.items[0].metadata.name}') -- env | grep "ARGOCD_"

# Check if Vault is also integrated
kubectl exec -n contextops $(kubectl get pods -n contextops -l app=contextops-gateway -o jsonpath='{.items[0].metadata.name}') -- env | grep "VAULT_"
```

## 📋 **5. FILTERED QUERIES FOR MONITORING**

### Multi-Environment Application Tracking
```bash
# Get all applications for acme-corp customer across environments
kubectl get applications -n contextops -l contextops.io/customer=acme-corp -o json | jq '.items[] | {name: .metadata.name, environment: .metadata.labels."contextops.io/environment", applicationset: .metadata.labels."contextops.io/applicationset"}'

# Applications in dev environment only
kubectl get applications -n contextops -l contextops.io/environment=dev -o json | jq '.items[].metadata.name'

# All ContextOps monitored resources
kubectl get applications -n contextops -l contextops.io/monitored=true -o json | jq '.items[] | .metadata.name'
```

### ApplicationSet Health and Status
```bash
# Check if ApplicationSet is generating applications successfully
kubectl get applicationsets -n contextops contextops-sample-apps -o json | jq '.status.conditions[] | select(.type == "ResourcesUpToDate")'

# Count of applications generated by ApplicationSet
kubectl get applicationsets -n contextops contextops-sample-apps -o json | jq '.status.resources | length'

# List all resources managed by ApplicationSet
kubectl get applicationsets -n contextops contextops-sample-apps -o json | jq '.status.resources[] | {kind: .kind, name: .name}'
```

## 🎯 **Quick Reference Commands**

```bash
# One-liner: ApplicationSet status
kubectl get applicationsets -n contextops contextops-sample-apps -o jsonpath='{.status.conditions[?(@.type=="ResourcesUpToDate")].status}'

# One-liner: Count generated applications  
kubectl get applications -n contextops -l contextops.io/applicationset=contextops-sample-apps --no-headers | wc -l

# One-liner: ContextOps health check
curl -s -u admin:admin http://45.55.118.17/health | jq -r '.status'

# One-liner: ArgoCD integration enabled?
kubectl get configmap contextops-integration-config -n contextops -o jsonpath='{.data.argocd\.enabled}'
```

## 📝 **Note on API Authentication**

- **ContextOps API**: Requires basic auth (`-u admin:admin`)
- **Kubernetes API**: Uses kubectl context/config
- **kubectl proxy**: No additional auth needed after proxy setup
- **GitOps endpoints**: May require customer context in request body
